# Adept Android IDE

A sleek, high-performance Integrated Development Environment (IDE) engineered exclusively for Android devices. Adept empowers developers to write, edit, and compile mobile application code natively on the go—transforming your tablet or smartphone into a dedicated, offline software development station. 

Designed specifically for the Android ecosystem, Adept provides a premium, untethered coding experience for building and refining mobile apps anywhere, without relying on cloud servers.

## ✨ Core Features

* **Offline Local Compilation:** Includes an embedded JDK and Kotlin compiler pipeline, allowing you to build, compile, and generate standard Android applications entirely on-device.
* **Advanced Syntax Engine:** Delivers flawless, high-fidelity syntax highlighting with intelligent pattern matching for Android-centric languages including Kotlin, Java, XML, and Groovy.
* **Mobile-First App Development:** Structured to handle standard Android project hierarchies seamlessly. Easily navigate layouts, manifests, and source code directly from your touch interface.
* **Premium Developer UI:** Engineered with a distraction-free, immersive dark mode (Darcula standard) designed to reduce eye strain during prolonged mobile coding sessions.
* **Modern Android Architecture:** Fully compliant with modern Android design standards, featuring a responsive code editor with auto-indentation and native adaptive launcher assets.

## 🚀 Installation & Downloads

You do not need to build the app from source to use it. You can download the pre-compiled, ready-to-use APK directly from the Releases page:

1. Navigate to the **Releases** section on the right side of this repository.
2. Download the latest `.apk` file attached to the release.
3. Install the APK on your Android device (you may need to allow installations from unknown sources in your tablet's settings).

## 🛠️ Building From Source

If you want to contribute, learn from the source code, or modify the IDE:
1. Clone this repository to your local Android environment (via Termux) or desktop IDE.
2. Build the project using your local Gradle pipeline.
3. *Note: This repository contains necessary local compiler assets (`jdk.zip`, `kotlinc.zip`). Ensure your Git environment supports pulling large files over 50MB.*
